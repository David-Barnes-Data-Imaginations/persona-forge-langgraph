You add voice activity detection (VAD) and a couple of rules, and it feels like “hands-free” voice chat. You’ve already done the hard parts (audio capture + Faster-Whisper + Piper). Now you just need a thin VAD layer and some UX glue.

Below is a practical menu of options, plus drop-in code to get you there.

What you need conceptually

VAD to say “someone is speaking” vs “silence/noise.”

Endpointing rules to decide when to:

start a recording (when speech begins), and

stop (after N ms of silence).

(Optional) buffering so the first phonemes aren’t clipped (pre-roll ~200–500 ms).

Noise safeguards (aggressiveness threshold, noise gate, or NS/AGC).

That’s it.


Silero-VAD via ONNX Runtime Web

More robust to noise; runs entirely in the browser with WASM/WebGPU.

Slightly heavier than WebRTC VAD, but still fast on modern machines.

Option C (server-side VAD gate)

Send low-bitrate PCM/Opus continuously to your server and run webrtcvad (Python) or Silero-VAD there to carve utterances. This is nice if you want exactly one place to tune thresholds and log audio decisions.

# server_vad.py
import webrtcvad

vad = webrtcvad.Vad()
vad.set_mode(2)  # 0-3 (3 = most aggressive)

# Feed 10/20/30 ms 16-bit mono PCM @16kHz frames:
is_speech = vad.is_speech(frame_bytes, sample_rate=16000)


Use a small state machine:

idle → pre_speech (collect pre-roll) → in_speech → maybe_silence (count silence ms) → endpoint (ship buffer to Faster-Whisper).

Minimal client-side example (no external model, energy-based VAD)

Tune the thresholds (ENERGY_ON, ENERGY_OFF, END_SILENCE_MS) and consider a calibration step: collect background noise for 1 second, compute mean energy, and set thresholds relative to that (e.g., on = mean*4, off = mean*2).

Backend sketch (WebSocket + Faster-Whisper per utterance)

This accumulates frames while the client is “speaking”, and on UTTERANCE_END it runs transcription:
```
# @Claude: We could just adapt this instead of a whole new file maybe?
# server.py
import asyncio, websockets, wave, io, numpy as np, json
from faster_whisper import WhisperModel
from datetime import datetime

model = WhisperModel("small", compute_type="float16")  # choose your model

async def handler(ws):
  current_frames = []
  async for msg in ws:
    if isinstance(msg, bytes):
      # PCM int16 mono 16kHz
      current_frames.append(np.frombuffer(msg, dtype=np.int16))
    else:
      try:
        data = json.loads(msg)
      except:
        continue
      if data.get("type") == "UTTERANCE_END":
        if not current_frames:
          continue
        pcm = np.concatenate(current_frames)
        current_frames = []

        # Write to in-memory WAV
        buf = io.BytesIO()
        with wave.open(buf, 'wb') as wf:
          wf.setnchannels(1)
          wf.setsampwidth(2)
          wf.setframerate(16000)
          wf.writeframes(pcm.tobytes())
        buf.seek(0)

        segments, info = model.transcribe(
          buf, language="en", vad_filter=True,
          vad_parameters=dict(min_silence_duration_ms=300)
        )
        text = "".join(seg.text for seg in segments).strip()
        await ws.send(json.dumps({"type":"TRANSCRIPT", "text": text, "ts": datetime.utcnow().isoformat()}))

async def main():
  async with websockets.serve(handler, "0.0.0.0", 8765, max_size=2**23):
    await asyncio.Future()

if __name__ == "__main__":
  asyncio.run(main())
```

Note: you can remove client VAD and rely solely on server VAD  by sending every frame; just add a server-side state machine (like the client logic) and call transcribe when your server VAD fires an endpoint.

- UX polish (to feel like “Claude Voice Chat”)
- Open-mic toggle: One switch to enter “listening mode”; show a subtle waveform/LED when speech is detected.
- Timeout: If no speech for N seconds, auto-exit open-mic or keep it idle (your choice).
- Barge-in: While TTS is talking, if VAD fires, duck TTS and pause it so the user can interrupt naturally.
- Beep cues: tiny start/stop earcons help users trust the system.


Practical tips

- Start with energy-based VAD + hysteresis (above). It’s 20 lines, easy to reason about, and works in quiet spaces.
- If you need robustness: switch to WebRTC VAD (Python webrtcvad or JS ports) or Silero-VAD (ONNX).
- Tune silence windows per language (fast vs slow talkers). 400–800 ms is a sweet spot for English.
- Add pre-roll (>=300 ms) or your first consonant will get clipped.
- Turn on browser constraints: { echoCancellation: true, noiseSuppression: true, autoGainControl: true }.
- For a “never sends silence” design, do VAD on the client; for “simpler troubleshooting,” do it on the server.
